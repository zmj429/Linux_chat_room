# Linux 聊天室项目

## 项目介绍

使用 C++ 编程语言，采用面向对象封装，实现的网络聊天室系统。

本项目是一个基于 TCP 协议的简单聊天室，支持多客户端同时连接服务器进行消息收发。

### 功能特性

- 多线程处理：服务器使用多线程处理多个客户端连接
- 消息收发：客户端可以发送和接收消息
- 简单易用：代码结构清晰，便于学习和扩展

### 项目来源

学习自蓝桥云课（原实验楼）的一个实战课项目。地址：`https://www.lanqiao.cn/courses/3573`

## 依赖项

本项目需要以下环境：

- **编译器**：支持 C++11 标准的 g++ 编译器
- **操作系统**：Linux/Unix 系统（使用了 POSIX 线程库）

## 编译和运行

### 编译项目

进入 src 目录并执行 make 命令：

```bash
cd src
make
```

这将同时编译客户端和服务端程序。也可以单独编译：

```bash
make client  # 只编译客户端
make server  # 只编译服务端
```

### 运行服务端

```bash
./server              # 使用默认配置（IP: 0.0.0.0，端口: 8080）
./server 192.168.1.100 8080  # 指定 IP 和端口
```

### 运行客户端

```bash
./client              # 使用默认配置（IP: 127.0.0.1，端口: 8080）
./client 192.168.1.100 8080  # 指定服务器 IP 和端口
```

### 清理编译文件

```bash
make clean
```

## 代码说明

### 服务端 (Server)

- 监听指定端口，等待客户端连接
- 每个客户端连接创建一个独立线程进行处理
- 接收客户端消息并在服务器端显示
- 支持多客户端同时连接

### 客户端 (Client)

- 连接到服务器
- 使用两个线程：一个用于发送消息，一个用于接收消息
- 输入 `exit` 可以退出程序

## 常见问题

1. **编译时找不到线程库**

   确保系统已安装 pthread 库：
   ```bash
   sudo apt-get install pthread
   ```

2. **端口被占用**

   检查端口是否被其他程序占用，或更换端口号。

3. **连接被拒绝**

   确保服务端已启动，并检查 IP 地址和端口号是否正确。
